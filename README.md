# Техническое задание

Необходимо разработать веб-приложение **Habit Tracker**, которое позволяет пользователю создавать привычки, отслеживать их выполнение и анализировать прогресс.

Приложение должно быть реализовано с использованием указанных технологий и **архитектуры Feature-Sliced Design (FSD)**.

Основная задача приложения — предоставить пользователю удобный инструмент для **формирования привычек и анализа их выполнения**.

---

# Цель приложения

Пользователь должен иметь возможность:

* создавать привычки
* отмечать выполнение привычки по дням
* отслеживать **streak (серию выполнения)**
* анализировать процент выполнения
* просматривать визуализацию активности
* получать базовую статистику

---

# Используемые технологии

Обязательные технологии:

* **Vue 3**
* **Pinia**
* **Vue Router**
* **JavaScript (ES6+)**
* **HTML5**
* **CSS / SCSS**
* **Vite**

Дополнительно допускается использование:

* **dayjs / date-fns** — для работы с датами
* **Chart.js / ApexCharts** — для визуализации статистики

---

# Архитектура проекта (FSD)

Проект должен быть реализован с использованием архитектурного подхода **Feature-Sliced Design**.

Основная цель FSD — разделение приложения по **функциональным слоям**, что повышает масштабируемость, читаемость и поддерживаемость кода.

Основные слои архитектуры:

### App

Инициализация приложения.

Содержит:

* глобальные провайдеры
* router
* инициализацию стора
* глобальные стили

---

### Pages

Слой страниц приложения.

Каждая страница представляет отдельный пользовательский сценарий.

Примеры:

* DashboardPage
* HabitPage
* NotFoundPage

---

### Widgets

Крупные UI-блоки, объединяющие несколько features или entities.

Примеры:

* HabitList
* HabitStatistics
* ProgressWidget

---

### Features

Слой пользовательских действий.

Каждая feature представляет **конкретную функциональность пользователя**.

Примеры:

* create-habit
* edit-habit
* toggle-habit
* track-habit
* change-theme

---

### Entities

Бизнес-сущности приложения.

Примеры:

* habit
* achievement
* progress

Entities содержат:

* модели данных
* store
* UI компоненты сущности
* бизнес-логику

---

### Shared

Переиспользуемый код, не зависящий от бизнес-логики.

Содержит:

* UI компоненты
* утилиты
* стили
* конфигурации
* composables

---

Согласно архитектуре FSD.

# Функциональные возможности

## Управление привычками

Пользователь может:

* создать привычку
* редактировать привычку
* удалить привычку

Каждая привычка содержит:

* название
* дату создания
* историю выполнения
* текущий streak
* лучший streak

---

## Отметка выполнения привычки

Пользователь должен иметь возможность:

* отметить выполнение привычки за текущий день
* отменить отметку
* просматривать историю выполнения

---

## Streak (серия выполнения)

Для каждой привычки необходимо рассчитывать:

* текущую серию выполнения
* максимальную серию выполнения

Серия считается, если привычка выполнялась **несколько дней подряд без пропусков**.

---

## Визуализация прогресса

Прогресс должен отображаться в виде **календарной сетки**, аналогичной активности GitHub.

Особенности:

* отображение последних **90 дней**
* выполненные дни подсвечиваются
* отображается дата при наведении

---

## Статистика

Для каждой привычки отображается:

* процент выполнения за неделю
* процент выполнения за месяц
* текущий streak
* лучший streak
* количество выполненных дней

---

## Темная тема (опционально)

Приложение должно поддерживать:

* светлую тему
* темную тему

Тема должна сохраняться в **LocalStorage**.

# Описание страниц приложения

---

# Главная страница (Dashboard)

Содержит:

* список привычек
* кнопку создания привычки
* краткую статистику

Каждая привычка отображается в виде карточки.

Карточка должна отображать:

* название привычки
* текущий streak
* кнопку отметки выполнения

---

# Страница привычки

Содержит:

* название привычки
* календарь активности
* статистику выполнения
* лучший streak
* текущий streak

Также пользователь может:

* редактировать привычку
* удалить привычку

---

# Страница 404

Если пользователь переходит по несуществующему маршруту:

* отображается страница **404 Not Found**
* должна быть кнопка перехода на главную страницу

# Работа с состоянием

Для управления состоянием используется **Pinia**.

Store должен содержать:

```
habits: []
completedDates: {}
achievements: []
```

Основные действия:

```
addHabit()
removeHabit()
updateHabit()
toggleHabitCompletion()
calculateStreak()
calculateCompletionRate()
```

---

# Работа с датами и статистикой

Приложение должно корректно работать с датами.

Необходимо реализовать:

### Подсчет streak

Алгоритм:

* проверяется последовательность выполненных дней
* если день пропущен — серия обнуляется

---

### Процент выполнения

Формула:

```
(количество выполненных дней / количество дней периода) * 100
```

Периоды:

* неделя
* месяц

---

# Хранение данных

Для хранения данных необходимо использовать:

**LocalStorage**

Необходимо реализовать:

* автоматическое сохранение состояния
* загрузку состояния при запуске приложения

Работа с LocalStorage должна быть вынесена в **shared/lib**.

---

# Общие базовые требования

* использовать единый **code-style**
* компоненты должны быть **переиспользуемыми**
* бизнес-логика не должна находиться в UI
* логика должна находиться в **entities или features**

---

# Требования к именованию

* компоненты — **PascalCase**
* функции и переменные — **camelCase**
* директории FSD — **kebab-case**

---

# Требования к стилям

* использовать **SCSS или CSS**
* не использовать inline-стили
* использовать переменные для: цветов, отступов, размеров

---

# Git требования

* использовать **GitFlow**

Основные ветки:

* main
* develop
* feature/*

---

Использовать **Conventional Commits**

Примеры:

```
feat: add habit creation
feat: implement streak calculation
fix: fix calendar rendering
refactor: move logic to entities layer
```

---

# Дополнительные требования

* код должен быть **чистым и читаемым**
* избегать дублирования кода
* использовать **computed**
* использовать **composition API**
* использовать **переиспользуемые composables**
